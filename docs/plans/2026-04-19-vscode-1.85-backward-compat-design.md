# VS Code 1.85.2 Backward Compatibility — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Analyze and implement VS Code version compatibility for the Semutssh Copilot extension, targeting the oldest possible VS Code version given the current architecture.

**Architecture:** The extension uses `vscode.lm.registerLanguageModelChatProvider` to integrate with VS Code Copilot Chat. This is a Proposed API that only became available in recent VS Code versions. We analyze which proposed APIs are used and which are available in which VS Code versions.

**Tech Stack:** VS Code Extension API (Proposed), TypeScript, Node.js

## Executive Summary

After analyzing the VS Code codebase git history, the following facts are established:

| Proposed API | Added to VS Code | First Available In |
|---|---|---|
| `chatProvider` | 9 April 2026 | VS Code 1.100+ (Insiders) |
| `languageModelCapabilities` | 9 April 2026 | VS Code 1.100+ (Insiders) |
| `languageModelSystem` | 9 April 2026 | VS Code 1.100+ (Insiders) |
| `languageModelThinkingPart` | 9 April 2026 | VS Code 1.100+ (Insiders) |
| `languageModelToolResultAudience` | 9 April 2026 | VS Code 1.100+ (Insiders) |
| `languageModelToolSupportsModel` | 9 April 2026 | VS Code 1.100+ (Insiders) |
| `languageModelProxy` | 9 April 2026 | VS Code 1.100+ (Insiders) |

**All proposed APIs used by this extension were added on the same day (April 9, 2026).** VS Code 1.85.2 was released ~November 2025. These APIs did not exist in 1.85.2.

**Conclusion:** There is no code path that makes this extension work in VS Code 1.85.2. The extension fundamentally depends on APIs that did not exist at that time.

## Technical Analysis (For Documentation)

### Why 1.85.2 Cannot Be Supported

The extension's core functionality — registering a language model chat provider — relies on:

```typescript
// extension.ts:38
vscode.lm.registerLanguageModelChatProvider(
    "semutssh",
    chatProvider as unknown as vscode.LanguageModelChatProvider
);
```

This requires:
1. `vscode.lm` namespace (added ~VS Code 1.85)
2. `LanguageModelChatProvider` interface (proposed API)
3. `LanguageModelChatInformation` for model discovery (proposed API)
4. `ProvideLanguageModelChatResponseOptions` for chat responses (proposed API)
5. All the additional proposed APIs listed in `package.json:93-100`

The oldest stable VS Code version that has ANY of these is likely 1.96+, but the specific APIs used (especially `chatProvider`) only appeared in the April 2026 proposed API files.

### What Would Need to Change

If a theoretical older API were available, the following files would need conditional logic:

| File | What Would Change |
|---|---|
| `package.json:8-9` | `engines.vscode` range |
| `package.json:22-25` | `enabledApiProposals` list |
| `src/extension.ts:38-41` | API availability check via `typeof vscode.lm !== 'undefined'` |
| `src/providers/semutsshProviderBase.ts` | Conditional type usage based on API availability |
| `src/providers/semutsshChatProvider.ts` | Conditional response processing based on API version |
